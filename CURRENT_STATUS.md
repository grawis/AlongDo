# AlongDo 目前現況

## 專案定位

AlongDo 目前正處於以下兩個階段之間：

- 使用模擬資料的 MVP 展示版
- 結合真實定位與 Google Maps 能力的原型版

目前的開發方向是：

1. 先完成真實定位與附近任務查詢
2. 再完成 Google Maps 路線規劃整合
3. 接著補強地圖呈現與互動
4. 最後再串接真實後端任務 API

---

## 目前 App 功能

### 1. 首頁

- 顯示個人任務、團隊任務與群組的摘要資訊
- 讓使用者快速掌握目前任務概況

### 2. 我的任務

- 顯示個人任務清單
- 目前資料由 `services/api.js` 的 `fetchPersonalTasks()` 提供

### 3. 團隊任務

- 顯示團隊任務清單
- 目前資料由 `services/api.js` 的 `fetchGroupTasks()` 提供

### 4. 群組管理

- 顯示目前群組清單
- 顯示群組名稱、成員數與啟用狀態

### 5. 新增任務

- 提供表單新增個人任務
- 呼叫 `createPersonalTask()`，並同步加入本地任務狀態

### 6. 附近任務 / 定位

- 嘗試取得裝置目前位置
- 呼叫 `fetchNearbyTasks(coords)` 查詢附近任務
- 當 `USE_MOCK_API = false` 時，可切換為 Google Maps Nearby Search 流程

### 7. 路線規劃

- 可選取多筆任務並模擬執行順序
- 當 `USE_MOCK_API = false` 時，`planRoute()` 已預留 Google Directions API 整合
- 可同時規劃個人任務與團隊任務

### 8. 天氣提醒

- 顯示天氣狀態與提醒文字
- 目前仍使用模擬資料
- 後續可替換成真實天氣 API

---

## 目前實作重點

- `App.js` 負責頁籤切換與初始資料載入
- `services/api.js` 是目前主要的資料抽象層
- `USE_MOCK_API` 可切換 mock 模式與未來真實 API 模式
- Google Maps 相關流程已預留在：
  - `fetchNearbyTasks()`
  - `planRoute()`
- `screens/NearbySimulationScreen.js` 已包含真實定位邏輯
- `screens/RoutePlannerScreen.js` 已支援多來源任務路線規劃

---

## 專案現在實際在做什麼

這個專案目前已不只是靜態 MVP 畫面。

它正在逐步往更接近真實使用情境的原型發展，包括：

- 可以取得使用者真實定位
- 可以根據座標查詢附近任務
- 後續可透過 Google Maps 進行路線規劃

簡單說：

AlongDo 正在從「模擬資料待辦 App」升級成「具備位置感知能力的任務助理原型」。

## 已知問題與缺口

### 1. 中文亂碼曾經存在

- 專案先前有多個檔案出現中文亂碼
- 目前主要 UI 文案已整理為繁體中文
- 但仍需要持續留意未使用檔案或舊檔是否殘留亂碼

### 2. 附近任務功能仍需穩定化

- 附近任務功能已經使用 `expo-location`
- 先前 Expo log 曾出現依賴解析或模組相容性問題
- 目前已先降級為可展示流程版本，但仍需實機或模擬器驗證穩定性

### 3. 部分畫面仍以展示為主

- 首頁、團隊任務、群組管理目前仍偏向展示型畫面
- 規格中的部分互動尚未完全做完

例如：

- 群組啟用 / 停用切換尚未完成
- 團隊任務狀態切換尚未完整落地
- 新增任務欄位仍比完整規格簡化

### 4. 仍有舊版或未使用檔案

- `screens/MapScreen.js`
- `screens/PersonalTasksScreen.js`

這些檔案目前沒有被 `App.js` 使用，較像舊版或過渡檔案。

## 建議下一步

在繼續新增功能前，先把基礎穩住：

1. 持續清理專案裡殘留的亂碼或舊文案
2. 驗證定位功能在 Android 模擬器或實機上可正常運作
3. 驗證 `USE_MOCK_API = false` 時附近查詢與路線規劃流程是否可用

完成後，再繼續推進附近任務與路線規劃主線。
