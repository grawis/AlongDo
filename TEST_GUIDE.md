# AlongDo 測試安裝與操作說明

這份文件提供給要協助測試 AlongDo 的同學或隊友，內容包含：

- Android 手機安裝 APK 的方式
- Android 模擬器安裝 APK 的方式
- 如何加入同一個群組
- 建議測試流程
- 常見問題

---

## 1. 測試前注意事項

AlongDo 目前是 Android 測試版 App，測試前請先注意：

- 這不是 Google Play 商店安裝的正式版本
- 需要使用 APK 檔案安裝
- 每台裝置第一次開啟 App 時，都會建立自己的 Firebase 身分
- 如果想測多人群組功能，必須使用邀請碼加入同一個群組

---

## 2. Android 手機安裝方式

### 2-1. 取得 APK

測試者會收到一個檔案：

- `app-release.apk`

你可以透過以下方式傳給對方：

- Line
- Discord
- Google Drive
- USB
- 任何檔案傳輸方式

### 2-2. 安裝 APK

1. 在 Android 手機上打開 `app-release.apk`
2. 如果手機跳出安全提示，請允許安裝未知來源應用程式
3. 按下 `安裝`
4. 安裝完成後打開 `AlongDo`

### 2-3. 如果手機顯示無法安裝

有些手機會要求你先開啟安裝權限，常見位置例如：

- `設定 -> 安全性 -> 安裝未知來源應用程式`
- `設定 -> 應用程式 -> 特殊應用程式存取 -> 安裝未知應用程式`

通常只要允許目前用來開啟 APK 的 app，例如：

- Chrome
- Google Drive
- 檔案管理員

就能繼續安裝。

## 3. Android 模擬器安裝方式

如果測試者使用 Android Studio 模擬器，可以用以下方式安裝。

### 方式 A：直接拖曳 APK

1. 先打開 Android 模擬器
2. 將 `app-release.apk` 直接拖進模擬器視窗
3. 等待自動安裝完成
4. 安裝完成後打開 `AlongDo`

### 方式 B：使用 adb 指令安裝

如果電腦已安裝 Android SDK，可在終端機輸入：

```bash
adb install -r app-release.apk
```

如果 APK 不在目前資料夾，也可以改成完整路徑，例如：

```bash
adb install -r D:\AlongDo\android\app\build\outputs\apk\release\app-release.apk
```

安裝完成後，在模擬器中打開 `AlongDo` 即可。

## 4. 安裝後如何加入群組

1. 打開 `AlongDo`
2. 進入 `群組` 頁面
3. 如果你是群組建立者：
   - 建立新群組
   - 記下畫面上的邀請碼
4. 如果你是加入者：
   - 在 `用邀請碼加入群組` 區塊輸入邀請碼
   - 輸入你在群組中要顯示的暱稱
   - 按下 `加入群組`

成功後，雙方就會進入同一個群組。

## 5. 建議測試流程

### 建立群組的人

1. 進入 `群組`
2. 建立一個新群組
3. 設定自己的群組內暱稱
4. 把畫面上的邀請碼傳給其他人

### 加入群組的人

1. 進入 `群組`
2. 在 `用邀請碼加入群組` 區塊輸入邀請碼
3. 輸入自己的群組暱稱
4. 按下 `加入群組`

### 建議一起測的功能

加入成功後，可以一起測：

- 建立個人任務
- 建立團隊任務
- 固定地址任務
- 彈性地點任務
- 路線規劃
- 天氣提醒
- 任務狀態切換
- 任務刪除
- 群組啟用 / 停用

## 7. 常見問題

### Q1：安裝了同一個 APK，為什麼還看不到同一個群組？

因為每台裝置都會有自己的 Firebase 身分，所以安裝同一個 APK 並不代表會自動進入同一個群組。  
需要使用邀請碼加入群組。

### Q2：為什麼重裝 App 後看不到原本資料？

因為目前使用的是 Firebase 匿名登入。  
若刪除 App、清除資料或換裝置，系統可能會建立新的匿名身分，需要重新加入群組。

### Q3：沒有 Google Play 商店也能裝嗎？

可以，這是直接安裝 APK，不需要透過 Google Play 商店。

### Q4：模擬器沒有定位怎麼辦？

請到 Android Studio 模擬器的：

- `Extended Controls -> Location`

設定座標後按 `SET LOCATION`，再回 App 重新抓取位置。
