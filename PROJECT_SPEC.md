# AlongDo 專案規格

## App 名稱

AlongDo

## 核心概念

AlongDo 是一款結合「地點」與「群組協作」的待辦 App。

傳統待辦 App 通常以時間提醒為主，
AlongDo 則以「使用者現在在哪裡」與「目前情境」來提醒任務。

目標是讓使用者在順路時完成事情，也能讓群組成員彼此協助。

標語：
到哪都順手完成更多事。

---

## MVP 開發目標

先用 React Native + Expo 做出可在 Android 上展示的原型 App。

第一版 MVP 先不要實作以下功能：

- 真實 GPS 地理圍欄
- 後端資料庫
- Firebase
- 真實天氣 API
- 真實地圖路徑最佳化

前期以 mock data 與模擬流程為主，
重點是能完整展示使用流程。

---

## 核心功能

### 1. 個人任務建立與地點綁定

使用者可以建立個人任務。

每筆任務至少包含：

- 任務名稱
- 地點名稱
- 地點類型
- 隱私設定
- 任務狀態

MVP 階段：

- 使用者可手動輸入地點名稱
- 先不要求真實 GPS 座標

範例：

- 買雞蛋
- 地點：全聯
- 地點類型：超市

---

### 2. 群組建立與團隊任務分享

使用者可以建立或加入群組。

在群組中，使用者可以建立共享任務。

群組成員可以查看並協助完成團隊任務。

MVP 階段：

- 先使用 mock 群組

群組範例：

- 家庭
- 室友
- 朋友

每筆團隊任務至少包含：

- 任務名稱
- 群組名稱
- 建立者名稱
- 地點名稱
- 任務狀態

---

### 3. 群組啟用 / 停用功能

使用者可以將加入的群組切換為啟用或停用。

當群組啟用時：

- 使用者可以收到該群組的提醒
- 使用者可以和該群組分享任務

當群組停用時：

- 暫時停止接收該群組提醒
- 避免無關通知干擾

MVP 階段：

- 每個 mock 群組提供一個 toggle 切換

---

### 4. 個人任務的地點提醒

當使用者接近個人任務地點時，系統應觸發提醒。

MVP 階段：

- 一開始可用模擬流程
- 目前實作方向已逐步加入真實定位能力

提醒方式範例：

- 顯示使用者已接近某個任務地點

---

### 5. 團隊任務的協助提醒

當群組成員接近某個團隊任務地點時，
系統可詢問他是否願意順手協助完成。

範例：

- 順路買東西
- 幫忙代拿物品
- 協助處理共用雜務

MVP 階段：

- 可先用模擬流程展示
- 後續再接真實定位判斷

---

### 6. 團隊任務狀態切換

團隊任務需支援狀態切換。

狀態流程：

1. `pending`
2. `in_progress`
3. `completed`

互動規則：

- 第一次點擊：`pending -> in_progress`
- 第二次點擊：`in_progress -> completed`

目的：

- 避免多位成員同時重複執行同一件事

MVP 階段：

- 實作在團隊任務卡片上

---

### 7. 多任務路線規劃

使用者可以選取多筆要完成的任務。

系統根據目前位置與任務地點規劃較方便的執行順序。

MVP 階段：

- 先不用做 GTSP 或真實最佳化演算法
- 可先做 mock 路線結果
- 目前已開始準備串 Google Directions API

需要包含：

- 可選取多筆任務
- 路線規劃按鈕或自動規劃結果
- 顯示一條模擬順序結果

範例：

目前位置 -> 全聯 -> 影印店 -> 五金行

---

### 8. 天氣情境提醒

系統可根據天氣資訊提供情境式提醒。

範例：

- 若降雨機率高，提醒帶傘

MVP 階段：

- 先不接真實天氣 API
- 使用 mock weather data

顯示範例：

- 天氣：下雨
- 提醒：今天降雨機率高，記得帶雨傘

---

## 建議畫面

### 1. 首頁

顯示：

- App 名稱：AlongDo
- 標語
- 導航入口：
  - 我的任務
  - 團隊任務
  - 群組
  - 附近任務
  - 路線規劃
  - 天氣提醒
  - 新增任務

---

### 2. 我的任務

顯示個人任務清單。

每張任務卡應顯示：

- 任務名稱
- 地點
- 地點類型
- 隱私設定
- 狀態

使用者可：

- 查看任務
- 變更狀態
- 選取任務加入路線規劃

---

### 3. 團隊任務

顯示團隊任務清單。

每張任務卡應顯示：

- 任務名稱
- 群組名稱
- 建立者名稱
- 地點
- 狀態

使用者可：

- 點擊一次設為 `in_progress`
- 再點一次設為 `completed`

---

### 4. 群組管理

顯示 mock 群組：

- 家庭
- 室友
- 朋友

每個群組應有：

- 群組名稱
- 啟用 / 停用切換

---

### 5. 新增任務

讓使用者建立 mock 任務。

欄位建議包含：

- 任務名稱
- 地點名稱
- 地點類型
- 個人或團隊
- 隱私設定
- 若為團隊任務則可選群組名稱

MVP 階段：

- 允許只存在本地 state

---

### 6. 附近任務

可展示：

- 目前位置取得流程
- 附近任務查詢結果
- 後續地圖呈現預留區塊

前期可先用模擬清單與提示訊息，
之後逐步接入真實定位與地圖畫面。

---

### 7. 路線規劃

顯示可選取的任務清單。

使用者選取多筆任務後，
顯示規劃順序或模擬路線結果。

範例：

目前位置 -> 全聯 -> 影印店 -> 五金行

---

### 8. 天氣提醒

顯示 mock weather 資訊：

- 天氣：下雨
- 降雨機率：80%
- 建議提醒：今天降雨機率高，記得帶雨傘

並提供按鈕載入或模擬提醒結果。

---

## 資料模型

### Task

每筆任務建議包含：

- `id`
- `title`
- `type`: `personal` 或 `group`
- `locationName`
- `locationType`
- `privacy`: `private` 或 `public`
- `status`: `pending`、`in_progress`、`completed`
- `groupName`
- `ownerName`
- `selectedForRoute`

### Group

每個群組建議包含：

- `id`
- `name`
- `enabled`

### Weather

mock weather 建議包含：

- `condition`
- `rainProbability`
- `reminderMessage`

## MVP 暫不處理項目

先不要實作以下功能：

- 真實 geofencing
- 背景定位追蹤
- Firebase
- 登入系統
- 真實地圖最佳化演算法

- DBSCAN
- GTSP 演算法

## 開發風格

請以小步驟方式逐步實作。

保持程式碼簡單、清楚、容易閱讀。

每次修改後，簡短說明：

- 建立或修改了哪些檔案
- 各個畫面如何運作
- 如何在 Android 模擬器中測試
